# Release Docs / 版本文档

## 中文

- 每个正式版本发布时，在该目录下新增对应版本子目录，例如 `docs/releases/v0.1.0/`。
- 版本目录至少包含：发布说明、迁移说明、当时的 API/兼容策略摘要。
- 当前正式发布快照：`docs/releases/v0.1.1/`
- 当前发布候选快照：`docs/releases/v0.2.0/`（论文复现可审计版本；正式 GitHub Release 以 tag 发布为准）

## English

- Create a matching subdirectory here for every formal release, for example `docs/releases/v0.1.0/`.
- Each version directory should contain release notes, migration notes, and the API/compatibility summary for that release.
- Current formal release snapshot: `docs/releases/v0.1.1/`
- Current release-candidate snapshot: `docs/releases/v0.2.0/` (auditable paper-reproduction version; the formal GitHub Release is defined by the published tag)
